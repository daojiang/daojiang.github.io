General Information

  • All stylesheets are located under assets/css. Main stylesheet is style.css
  • Styles written in LESS assets/less/style.less and compiled to CSS using a free app called LESS.app
  • All scripts are located under assets/js. Theme specific script is theme.js
  • Replace favicon.png with a square .png favicon of your own (32x32 or 64x64)
  • Place your logo within .navbar-brand
  • Custom background overlay patterns for .background-overlay can be found under images/patterns
  • Worx is built on the popular front-end-framework Bootstrap 3. If you're already familiar with Bootstrap customizing this theme will be a breeze. But even if you've never heard of Bootstrap before, don't worry, as Bootstrap is very easy to understand and has a great documentation: http://getbootstrap.com/getting-started/

Creating your own styles

IMPORTANT: If you want to add custom styles don't edit the main stylesheet, use custom.css located under assets/css to add your own styles, as the main stylesheet style.css will be overridden by future updates.

custom.css already includes the style for the default parallax background images. By changing the value of only four properties (where it says: /* Replace with your primary color */) you can create your very own color scheme in less than a minute.

For some color inspiration head over to kuler.adobe.com or pltts.me

Page Loading Overlay

As a portfolio theme with plenty of photos and maybe even some videos your site can become quite big in size. That's why we'll show our visitor a fullscreen overlay, while all the resources are getting downloaded in the background. Right after the opening body tag we add the following piece of code: 

<div id="loadbox">
	<div class="wrapper">
		<div class="inner">
			<!-- Here goes your custom preload content -->
		</div>
	</div>
</div>

This overlay will fade out once the page is fully loaded. By default it only shows a white background with a spinning wheel. The spinner is not a graphic. It is a jQuery plugin called spin.js that will create a unique spinning wheel. You can set every detail of the spinner by yourself, creating a unique loading wheel. It's a detail, but hey, this is the first impression a visitor gets of your site, and we want to make sure, it's a good one ;-)

Head over to our main script assets/js/theme.js and at the very top you will find the settings for two spinning wheels.

Parallax: Background Images

Ian Lunn
http://www.ianlunn.co.uk/plugins/jquery-parallax/

Background images are scrolling slower than the rest of the page, creating the impression of depth on your website. We are using this effect on the home page, and the three sections testimonials, tweets and quote that work as a divider between the white background sections.

By default all background images scroll at half speed. You can customize the scroll speed of each background image. Therefor go to assets/js/theme.js to the very bottom and replace the 0.5 values with a number between "0" and "1". The value has to be lower than "1", as "1" is the default scrolling speed, where you don't notice any parallax effect.

Example Code: Custome Parallax Scroll Speed

$('#home').parallax("50%", 0.5); 				// Background: Home
$('#testimonials').parallax("50%", 0.5); 		// Background: Testimonials
$('#tweet').parallax("50%", 0.5); 				// Background: Tweets
$('#quote').parallax("50%", 0.5); 				// Background: Quote

To add your own parallax sections create a section like the one in the example below and fill it with your own content. The .background-overlay container is optional, but recommended, as it can be hard to read text on a background image.

Example Code: Parallax Section

/* Stylesheet Example Code for Background Image */
<style>
#section-id {
	background-image: url("path/to/your/backgroundimage.jpg");
}
</style>

<section id="section-id" class="parallax">
	<div class="background-overlay"></div>
	<div class="container">
		<!-- Here goes your content of this parallax section -->
	</div>
</section>

// Custom Parallax Scroll Speed for this section
<script>
$('#section-id').parallax("50%", 0.25);
</script>

The default section background images are already included within custom.css. Simply update the path to link to your own images.

Section: Home

To set the full-screen background image of your homepage go to assets/css/custom.css and enter the URL to your custom background for #home.

.background-overlay adds a light dark overlay and a background pattern of your choice on top of your background images. As text placed on a plain background image is often hard to read. Worx comes with a total of 20 background overlay patterns. You'll find them under images/patterns. Add a container of .background-overlay to your .parallax sections as well, to make the text on top of your background images more readable.

<section id="home" class="parallax">

	<div class="wrapper">
		<div class="inner">
			<!-- Here goes your content -->
		</div>
	</div>	
	<div class="home-btn-continue">
		<a href="#about" class="btn-light animated bounceInRight">More</a>
	</div>
</section>

Make sure to use the .wrapper and .inner container as shown in the example above to align your home content vertical centered. Adding the class of .animated and one of the 50+ animate.css effect classes will make the specific element interactive. Fading in, boucing in or whatever effect you want to apply. In our exapmle above the "more" button will bounce in from the right. To learn more about the CSS animations read on..

CSS Animations

To bring your homepage alive we use the popular animate.css effects library. Choose from a total of 50+ effects. You can apply an effect to every element on your home page by ading a class of .animated and one of the effects classes, such as .fadeInUp or bounceIn.

By default CSS animations are set to a duration of 1s and a delay of 0s. To change the duration and delay for a single effect you have to open your custom.css stylesheet add a class of .animated with your custom setting, as seen in the example below.

Example Code: Custom CSS Animation

#home .animated {
  -webkit-animation-duration: 1.5s;
  animation-duration: 1.5s;
  -webkit-animation-delay: 3s;
  animation-delay: 3s;
}

The example above will cause all animations within #home to be executed with a length of 1.5s and a delay of 3s.

Twitter

Adding your most recent tweets within the #tweet container requires some work on your end, but if you follow the outlined steps it is fairly easy. We use James Mallison's Twitter API PHP plugin to output our latest tweets.

(1) Follow the instructions over at: http://stackoverflow.com/questions/12916539/simplest-php-example-for-retrieving-user-timeline-with-twitter-api-version-1-1/15314662#15314662 to create your Twitter Developer account.

Once you have got your consmumer key, consumer secret, access token and access token secret open tweet.php. On line 8 - 11 you will find the following piece of code: 

$settings = array(
'oauth_access_token' => "YOUR_OAUTH_ACCESS_TOKEN",
'oauth_access_token_secret' => "YOUR_OAUTH_ACCESS_TOKEN_SECRET",
'consumer_key' => "YOUR_CONSUMER_KEY",
'consumer_secret' => "YOUR_CONSUMER_SECRET"
);

Replace the four placeholders with the tokens from your Twitter Developer Account.

You're almost there. Last step is to tell the plugin which twitter account you want to pull tweets from and how many of the latest tweets from this account you want to show. You only need to update the following two values on line 15 and 16 with your own. 

$screenname = "ev"; // The screen name of the user for whom to return results for.
$count = 3; // Specifies the number of tweets to try and retrieve, up to a maximum of 200 per distinct request

If the number of tweets you entered is higher than one your tweets will be shown in a fading slideshow. The example above would output the three latest tweets from Twitter founder Evan Williams http://twitter.com/ev

Portfolio: Isotope v1.5.25

David DeSandro
http://isotope.metafizzy.co/

Creating a portfolio is a simple and straight forward process of two (to three) steps. (0) Optional: Create a portfolio filter. (1) Add portfolio items. (2) Create project pages.

Step 0: Create a Portfolio Filter

This step is optional. You can skip it, if you don't want to make your portfolio sortable.

Within the #portfolio section create an unordered list with an ID of #portfolio-filter. Add a single filter by creating a list item plus anchor with a data-filter attribute. This data-filter attribute holds the filtername. Add your filtername with a prepending dot "." Without the dot, the filter won't work. Create as many filter as you need. Add a class of .active to the list item that contains the filter, that you want to select by default. To learn more about the filtering visit http://isotope.metafizzy.co/docs/filtering.html

Example Code: Portfolio Filter

<ul id="portfolio-filter" class="list-inline">
  <li class="active"><a href="#" data-filter="*">All</a></li>
  <li><a href="#" data-filter=".filter1">Filter 1</a></li>
  <li><a href="#" data-filter=".filter2">Filter 2</a></li>
  <li><a href="#" data-filter=".filter3">Filter 3</a></li>
</ul>
Step 1: Add Portfolio Items

Each list item of #portfolio-list contains one portfolio item.

Example Code: Portfolio with Item

<ul id="portfolio-list">
	<li class="filter1 filter2">
		<img src="path/to/your/projectimage.jpg" alt="" />
		<div class="portfolio-item-content">
			<span class="header">Project Title</span>
			<p class="body">Short Project Description</p>
		</div>
		<a href="#project1"><i class="more"></i></a>
	</li>	
</ul>

The project link must point to the project page container, that we will create in the next step. Otherwise the project page won't open.

By default you open a project page by clicking it's more icon. If you want to make the entire list item clickable, wrap the anchor around the img and .portfolio-item-content.

Optional: In case you are going to use the portfolio filter, this is the time to assign filter(s) to your project by adding them as classes to the list item. In our example above we assign the two filters "filter1" and "filter2" to #project1.

Step 2: Create a Project Page

Add the following container to the #projects section. 

<div id="project1">
	<h3 class="title">Project Title</h3>
	<!-- Here goes your project content -->
</div>
